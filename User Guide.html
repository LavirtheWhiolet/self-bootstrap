Input Grammar

Input grammar consists of:

  • Rules (at least one)

  • Ruby code insertions (optional)

Rules

Rules have the following form:

nonterminal <- expression ;

You may use “=” or “←” instead of “<-”.

Ruby code insertions

Every Ruby code insertion has one of the following forms:

  • { code }

  • {... code ...}

  • {... code }...

They are written into the output file “as is”. A code insertion before all rules and other code insertions is put into the beginning of the output file. A code insertion after all rules and other code insertions is put into the end of the output file. Other code insertions are put somewhere in the middle of the output file.
